About

    做这个网站并没有特别明确的目的性，只是感觉工作以后自己的视野反而变得越来越窄了。可能在某个特定的领域会越来越深入，但是对全局上的了解有点匮乏。

    从技术层面来说，在一个具有一定规模的公司里，分功会比较明确。有的团队负责架构设计开发，有的团队开发各种库和组件，有点团队开发框架和构建工具，还有的团队负责开发应用等等。

    对于个人来说，每个团队之间都是完全独立的，除了自己所属团队的项目以外，其它的部分基本就是一团迷雾。

    当然，在现实工作中，并不需要你去了解一个大型应用开发过程中的所有技术环节。但是对于我来说，编程并不仅仅只是工作，也是我的一个爱好。就像是玩一款经营模拟类游戏，当你造出一座完整的城市，心里会有很强的满足感。所以我不想只了解其中微小的一部分，我想要尝试着去掌握一个完整的程序开发出来的全过程，从架构到前后端开发到数据库等等等等。可能并不足够专业，但至少掌握基本的流程和需要应用到的技术。

    所以仅以此网站作为自己练手的工具，开发一些自己觉得有意思的东西，其中会尝试用一些比较前沿时髦的新技术~ 踩一踩坑，锻炼一下自己技术层面的能力。

    网站使用原生ts和js，不使用任何框架，从底层开始开发。会用一些外界的工具和库，但能不用就尽量不用。

代码地址：https://github.com/ZhouyangYe/atat

Frontend

前端

前端准备做八个模块：admin、algorithms、common、hahaha、home、intro、notFound、pingpong。

构建工具：Webpack

相关技术：TypeScript，ES6+语法，LESS，HTML

Admin

    admin模块主要用来管理网站的一些资源：图片管理、页面展示、登陆等功能，是后台操作页面。
    
    技术方面的话会用到stream、buffer、pipe之类的数据传递技术，还有session、cookie、password加密之类的账号相关技术等。

Algorithms - 前往页面

    algorithms模块准备做成一个文件夹，类似Windows系统那样的文件夹，用来管理自己写的算法，以后会多刷一刷力扣上的题上传到这个algorithms模块。

    技术方面会用到树形数据结构和一些节点相关的操作，还可能用到类似highlight.js这样的库来给代码做高亮。
    
    计划在GitHub上开一个新的algorithms仓库，算法代码统一提交到这个仓库，然后页面调用GitHub接口获取这些文件展示在我自己的网站上。

Common

    common模块并不是一个页面，是一个库，里面用来放一些各个模块都会用到的通用方法，比如ajax、一些dom操作的封装、通用样式等等。

    common是以本地包的形式被安装到node_modules，然后被其它的模块以包的形式引入，以npm link的方式开发。因为每个模块之间都是独立的，从构建角度来说，每个模块都有自己的webpack配置，只能用这种方法来抽离通用逻辑。

Hahaha - 前往页面

    hahaha模块准备做成一个SPA，因为不知道该叫什么所以随便起了个名字。
    
    这个页面的设想比较复杂一些，准备做成类似微服务那样的架构，可以自己在上面开发和安装各种应用。

    另外，希望能通过这个页面沉淀出一个类似react、angular那样的前端框架，当然不可能像那些知名框架一样全面，开发一个成熟的框架工作量很大，不是一个人可以轻易完成的。

    其中的技术主要包含各种dom操作，路由等。难点在于前后端架构设计。

Home - 前往页面

    home模块打算做成一个3D的“家”的模型，里面可以放各种物件，家具之类的，还有和其它页面联动的功能。
    
    技术方面主要用到css3的3D功能（以后可能会考虑使用webgl，因为我也在学习开发游戏引擎）。

Intro

    intro模块就是我们当前所在页面，算是一个欢迎页面。

NotFound - 前往页面

    notFound顾名思义，就是在页面不存在的情况下展示的页面，里面打算放些彩蛋之类的东西，比如贪吃蛇小游戏或者俄罗斯方块之类。

Pingpong - 前往页面

    pingpong模块准备用来做一个打乒乓的网络游戏，这个会在服务器上单独开一个端口给socket.io使用。
    
    我已经用socket.io-client、socket.io、html和css做了一版，用户可以进行自由匹配对战，但是这个版本有很多缺陷，打算换成canvas重新开发一版，然后在此基础上加些新的功能比如抽球，旋转等等。

Backend

后端

后端用原生nodejs开发，基本结构已经搭好了，主要功能包括路由绑定，middleware绑定，handler绑定等等，很多地方借鉴了expressjs的设计，以后还有很多要改进的地方，比如session登陆功能，还有升级到http2等等.

前端模块级别的路由是由后端来处理的，每个模块都是一个SPA，模块之内的路由由前端来掌控。

基础结构：

resources

    resources文件夹用来存储所有动态和静态资源，像图片、视频等文件会被放在这里。

core

    core文件夹用来存放后端的主要逻辑（也就是路由绑定的handlers），get、post方法包括middleware的实现都在这里。

router

    router文件夹用来存放路由的绑定，以及绑定的实现方法。

apps

    apps文件夹存放所有的前端文件。

sql

    sql文件夹存放数据库相关脚本文件。

其它

    还有些文件，比如config文件存储数据库信息、服务器信息、开发模式等，还有一些自己写的工具：logger、支持绝对路径的require等等就不详细说了。

Dev Tools

开发小工具

scripts文件夹用来放一些代码管理层面的小工具：比如基于webpack封装了一个编译工具，等等。

    小工具是一些通过指令行来执行的脚本，用来编译代码，移除文件等操作。有自己设计的简单的UI，配合npm的package.json的scripts使用。
          
    webpack_common文件夹存放了所有前端模块通用的一些webpack配置。
